# 第19课：BP写作与路演

阶段五：融资发展第19课 / 共21课

> **本课目标**
> 商业计划书（BP）是融资的"敲门砖"。投资人每天收到大量BP，大部分连看都没看就被筛掉了。本课将教你投资人怎么看BP、BP的核心目标是什么、9页标准BP的每一页怎么写，以及如何做一场3分钟打动投资人的路演。

## 一、投资人怎么看BP？

### 投资人看BP的真实场景

理解投资人看BP的场景，才能写出投资人愿意看的BP：

| 场景特征 | 对你的启示 |
|---|---|
| 非常碎片化——开会时看、聚餐时看、路上用手机看 | BP必须能在手机上清晰阅读 |
| 大多数时间用手机看，不是电脑 | 大字大图，不要小字体 |
| 每天收到大量BP，先由助理/实习生初筛 | 标题和第一页要足够吸引人 |
| 一份BP花的时间不超过3分钟 | 重点信息要在前几页呈现 |
| 朋友推荐的BP直接到合伙人手里；公开渠道的BP可能被助理筛掉 | 尽量通过人脉推荐，不要冷投 |

> **常见错误**
> 字小又多、手机看不清；文件太大下载慢；把发给客户的产品手册当BP发。这些都是融资路上的"自杀行为"。

### 什么样的BP能引起投资人兴趣？

| 原则 | 说明 | 反面案例 |
|---|---|---|
| **定位要尖、要垂直** | 不能说"我是一个平台"，要说具体解决哪个具体问题 | "我们是一个企业服务SaaS平台"（太宽泛） |
| **趋势好、空间大** | 如AI、短视频、硬科技等。"趋势好、空间大、竞争少、增长快"四条中能证明两三条就很好 | 在一个萎缩的行业里做微创新 |
| **简单清晰** | 把事情说简单、把账算明白、把模型捋清晰 | 用了大量行业术语和技术细节 |
| **团队亮眼** | 大厂背景、厉害的经历、与事情匹配度非常高 | 团队经历与当前做的事毫无关系 |

## 二、BP的核心目标

> **核心原则：BP不是讲清所有业务，是引起兴趣。**
> 投资人对BP的期望不是"看完就完全理解你的业务"，而是"看完后想约你聊一聊"。BP只是激发兴趣的工具，深入的了解发生在后续的面谈中。

### BP写作的三条铁律

| 铁律 | 含义 | 实操建议 |
|---|---|---|
| **BP是业务逻辑的外在表现** | 描述不好要么是业务逻辑不清楚，要么是表述逻辑不清楚 | 写作前先用思维导图梳理业务逻辑 |
| **用最简洁的语言** | 字越少越好，多用数字和图表，多用业务模型 | 能用图表的不用文字，能用数字的不用形容词 |
| **讲人话** | 不要讲太多专业细节，讲商业核心 | 医疗项目不讲分子结构，讲"能治什么病、多少人会得、市场多大" |

## 三、BP标准结构详解（9页核心）

| 页码 | 内容 | 核心要点 |
|---|---|---|
| P1 | 封面 / 一句话定位 | Logo + 企业定位（一句话说清你是谁） |
| P2 | 用户痛点 | 场景化描述真实痛感 |
| P3 | 解决方案 | 产品如何解决痛点，用对比展示效果 |
| P4 | 用户证言 / 业务数据 | 快速增长曲线 |
| P5 | 运营数据 | 下载量、用户数、营收、转化率等 |
| P6 | 商业模式 + 市场规模 | 如何赚钱，收入公式 |
| P7 | 竞争分析 | 竞品对比矩阵 |
| P8 | 团队介绍 | 头像 + 职位 + 关键经历 |
| P9 | 融资目标与用途 | 资金分配饼图 + 增长曲线 |

### P1 封面：一句话定位

这是投资人对你的**第一印象**。两种写法：

| 写法 | 说明 | 例子 |
|---|---|---|
| **对标法** | 用知名公司做类比，让投资人秒懂 | "篮球版的抖音"、"企业服务领域的淘宝" |
| **描述法** | 客群 + 模式 + 属性 | "为初创企业提供一站式企业服务平台的SaaS公司" |

### P2 用户痛点

痛点要**场景化**。让投资人产生代入感：

| 类型 | 定义 | 例子 |
|---|---|---|
| **痛点（真痛）** | 不解决会很难受，用户愿意付费解决 | 半夜孩子哭闹等10分钟冲奶粉——快速温奶器是刚需 |
| **痒点（伪痛）** | 能忍，不着急解决 | 喝水等10分钟——可以忍，不构成购买动力 |

### P3 解决方案

用**戏剧性对比**展示效果：

**对比公式：**"原来折腾半小时~1小时 → 现在1分钟搞定，效率提升10倍，价格还便宜3/4"

解决方案页面不是技术说明书，而是展示"用了我们之后世界变得多好"。可以放产品截图、使用前后对比、关键数据。

### P4 用户证言

真实用户的正面反馈是最有说服力的。可以展示：

- 朋友圈截图、聊天记录

- 感谢信或感谢邮件

- 用户夸你的短视频截图

- NPS（净推荐值）评分

- 客户Logo墙（如果已有知名客户）

### P5 运营数据

展示**快速增长曲线**——投资人最看重的是增长趋势，不是绝对值。

| 数据维度 | 推荐展示方式 |
|---|---|
| 用户数 | 月度增长曲线（MAU） |
| 下载量 | 累计下载趋势 |
| 营收 | 月度/季度营收增长 |
| 转化率 | 免费→付费转化漏斗 |
| 留存率 | 次日/7日/30日留存 |
| 客单价 | 平均交易金额 |

### P6 商业模式 + 市场规模

**商业模式：**用收入公式说清楚怎么赚钱：

| 模式 | 收入公式 | 例子 |
|---|---|---|
| **B2C** | 收入 = 付费用户数 × 单价 × 利润率 | 知识付费APP：10万付费用户 × 199元/年 |
| **B2B** | 收入 = 客户数 × 客单价 × 续费率 | SaaS：100家企业 × 1万/年 × 90%续费 |
| **B2B2C** | 收入 = 合作B端数 × 每个B端用户数 × 分账比例 × 定价 | 平台模式：10个渠道 × 1万用户/渠道 × 30%分成 |
| **广告** | 收入 = 用户数 × 广告点击率 × CPC | 内容平台：100万DAU × 2%点击率 × 5元 |

**市场规模：**用权威数据支撑（第三方报告、行业白皮书），展示TAM（总可寻址市场）、SAM（可服务市场）、SOM（可获得市场）三层结构。

### P7 竞争分析

用**竞品对比矩阵**展示你的定位。常见格式：

| 维度 | 竞品A | 竞品B | 竞品C | **我们** |
|---|---|---|---|---|
| 价格 | 高 | 中 | 低 | **中** |
| 功能完整度 | 高 | 中 | 低 | **高** |
| 用户体验 | 中 | 高 | 中 | **高** |
| 市场覆盖 | 全国 | 一线城市 | 海外 | **全国（快速扩张）** |

> **技巧：**
> 把竞品放在一个二维坐标系中（如X轴=价格，Y轴=功能），把自己放在**第一象限**（高功能 + 合理价格），视觉上突出优势。

### P8 团队介绍

关键原则：**经历必须能支撑胜任力。**

| 要做 | 不要做 |
|---|---|
| - 头像 + 姓名 + 职位 + 关键经历 - 经历与当前岗位高度相关 - 突出过往成就（如"前字节跳动产品总监，负责XX产品从0到100万用户"） - 展示团队互补性（技术+产品+运营+商务） | - 只写名字和职位，没有经历 - 经历与当前岗位不符（如北大文学硕士来做CTO——只写"北大硕士"） - 团队背景高度同质化（全是技术、全是销售） |

### P9 融资目标与用途

这一页要回答三个问题：

1. **融多少钱？**给出合理的估值范围。估值不要报太高让人砍价，给出合理区间即可。

2. **钱花在哪？**用饼图展示资金分配（产品研发、市场推广、团队扩张、运营等比例）。

3. **花完带来什么？**必须有**增长曲线**——让投资人知道钱能带来什么样的回报。

P10（可选）：**联系方式**——微信 + 手机号。很多人会忘记加！

## 四、手把手写BP：准备工作到成稿

### 准备工作

1. **梳理业务逻辑：**用思维导图把用户、产品、市场、竞争、财务全部梳理一遍

2. **收集数据：**用户数据、财务数据、市场数据

3. **研究竞品：**至少分析3~5个直接和间接竞品

4. **明确融资需求：**算清楚需要多少钱、花在哪、能撑多久

### 写作步骤

| 步骤 | 内容 | 产出 |
|---|---|---|
| 第一步 | 写出每一页的核心标题（一句话） | 每页的标题句 |
| 第二步 | 围绕标题补充关键论据和数据 | 每页的要点和数字 |
| 第三步 | 设计视觉——把文字转化为图表和模型 | 可视化初稿 |
| 第四步 | 请合伙人/朋友试读——能否3分钟看懂 | 反馈意见 |
| 第五步 | 修改迭代，控制总页数不超过15页 | 终稿 |

### BP设计原则

- **大图大字：**手机能看清是第一原则

- **配色简洁：**不超过3种颜色

- **标题必须是句子：**不要写"团队介绍"，写"我们的团队来自字节跳动和律所的跨界组合"

- **一页一个核心信息：**不要试图在一页塞下太多内容

- **文件大小：**控制在10MB以内，方便邮件发送和手机下载

> **标题即句子技巧：**
> 标题写成完整句子有两个好处——①紧张忘词时读一遍标题就不会差太多；②投资人走神回来瞟一眼就知道重点，不需要从头看起。

## 五、三分钟路演技巧

### 路演的真实场景

- 一两百个投资人挤在一个闷热的屋子里

- 投资人可能在玩手机、回消息、走神

- 十几个项目车轮战，信息密度非常大

- **投资人30秒就能判断项目是否值得听**

### 路演PPT准备原则

| 原则 | 说明 |
|---|---|
| **大图大字** | 字要够大，远处的人也能看清 |
| **配色简洁** | 不超过3种颜色，保持品牌一致性 |
| **标题必须是句子** | 不要写名词标题，写完整句子 |
| **准备3分钟逐字稿** | 每分钟约250字，总计约750字，反复练几十遍 |

### 电梯演讲公式（30秒版）

**30秒模板：**  
"我们为【目标用户】解决【具体痛点】，通过【解决方案】，实现了【关键数据】。我们团队来自【背景】，现在需要【融资金额】来【用途】。"

### 路演心态

**路演不是让投资人决策的，是激发兴趣的。**激发兴趣 → 一对一聊天 → 深聊 → 投资。不要指望路演现场就拿到钱。

> **投资人心理洞察**
> 投资人像"渣男渣女"——吃着碗里的看着锅里的。但他们**更怕错过好项目（比投错更难受）**。可以利用这个心理：让投资人觉得"不看会错过"而不是"看了可能会亏"。

## 六、跟进投资人的方法

### 跟进三大原则

| 原则 | 说明 |
|---|---|
| **全力以赴，速战速决** | 在市场上待太久没融到钱，投资人会觉得"这项目没人要"——越拖越被动 |
| **能多拿就多拿** | 行业黑天鹅太多，多拿钱增加安全垫 |
| **不需要马上融资就先混个脸熟** | 让投资人知道你在做什么，未来融资时更容易推进 |

### 跟进方式

| 方式 | 频率 | 内容 |
|---|---|---|
| 定期更新进展 | 1~2周一次 | 新客户、新产品、新团队成员、关键数据增长 |
| 重要节点通报 | 关键事件时 | 重大合作、产品上线、融资里程碑 |
| 主动给出时间压力 | 必要时 | "现在估值1亿，下个月2亿，因为我们即将完成业务高峰" |

### 常见误区

> **专家型创业者陷阱：**
> 总想教育投资人，把投资人当学生。投资人不想成为行业专家，只想知道市场大不大、需求痛不痛、能不能赚钱。用类比（如"跟他类似，只是从线下变成了线上"）比讲技术参数有效得多。

### 投资人为什么不理你？

- **还在观望：**对你感兴趣但想再看看数据

- **真的忘了：**每天见太多项目，不是针对你

- **不感兴趣：**委婉地用不回复表达拒绝

无论哪种情况，定期更新进展都不会错。如果持续3个月没有回复，可以把精力转向其他投资人。

## 课后思考题

### Quiz：检验你的BP和路演准备

1. 用一句话（25字以内）写出你公司的定位。试试用对标法和描述法各写一个版本。

2. 你的用户最痛的点是什么？是真痛还是痒点？你愿意为这个痛点付费吗？

3. 画出你的商业模式的收入公式。公式中的每个变量目前是多少？6个月后目标是多少？

4. 列出你的3个主要竞品，用矩阵对比你们的差异。你的核心竞争力是什么？

5. 准备一段30秒的电梯演讲，说给身边的朋友听——朋友能听懂吗？
